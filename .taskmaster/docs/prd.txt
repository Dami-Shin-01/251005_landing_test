# 웹디자이너 포트폴리오 랜딩페이지 프로젝트

## 프로젝트 개요
야놀자 그룹의 디자인 시스템을 기반으로 한 전문 웹디자이너의 상세페이지 상품 소개 랜딩페이지를 개발합니다.

## 목표
- 야놀자 디자인 시스템을 참고한 고품질 포트폴리오 랜딩페이지 구축
- 반응형 디자인으로 모든 디바이스에서 최적의 사용자 경험 제공
- 애니메이션과 인터랙션을 통한 몰입감 있는 포트폴리오 전시

## 주요 기능 요구사항

### 1. Hero Section
- 풀스크린 히어로 섹션
- 대형 타이포그래피 및 그라디언트 텍스트 효과
- CTA 버튼 (View Work, Get in Touch)
- 스크롤 인디케이터 애니메이션
- 배경 그라디언트 애니메이션 효과

### 2. About Section
- 디자이너 소개 텍스트
- 통계 카운터 애니메이션 (프로젝트 수, 클라이언트 수, 경력 년수)
- 2단 그리드 레이아웃 (텍스트 + 통계)
- 스크롤 애니메이션 (AOS)

### 3. Services Section
- 3개 서비스 카드 (UI/UX Design, Web Design, Branding)
- 카드 호버 효과 (틸트, 섀도우)
- 아이콘 + 제목 + 설명
- 그리드 레이아웃 (자동 반응형)

### 4. Portfolio Section
- 프로젝트 그리드 레이아웃
- 이미지 + 오버레이 효과
- 호버 시 프로젝트 정보 표시
- 이미지 확대 애니메이션
- 최소 3개 프로젝트 전시

### 5. Achievements Section
- 4개 성과 카드 (프로젝트 완수, 클라이언트, 수상 경력, 고객 만족도)
- 숫자 카운터 애니메이션
- 그리드 레이아웃
- 카드 디자인

### 6. Contact Section
- 2단 레이아웃 (연락 정보 + 문의 폼)
- 이메일, 전화번호 표시
- 문의 폼 (이름, 이메일, 메시지)
- 폼 유효성 검사
- 제출 버튼

### 7. Navigation
- 고정 내비게이션 바
- 스크롤 시 자동 숨김/표시
- 모바일 햄버거 메뉴
- 스무스 스크롤
- 활성 섹션 하이라이트

## 디자인 시스템

### 색상 팔레트
- Primary: #FF385C (핑크)
- Secondary: #00D9FF (사이안)
- Accent: #A855F7 (퍼플)
- Dark: #0F0F23, #1A1A2E, #16213E
- Light: #FFFFFF
- Gray Scale: #F7F7F7 ~ #171717

### 타이포그래피
- 영문: Inter (Google Fonts)
- 한글: Noto Sans KR (Google Fonts)
- 크기: 12px ~ 72px (반응형)
- 무게: 300, 400, 500, 700, 900

### 간격 시스템
- 8px 기반 간격 시스템
- 0.5rem ~ 8rem

### Border Radius
- Small: 6px
- Medium: 8px
- Large: 12px
- XL: 16px
- 2XL: 24px
- Full: 9999px

## 기술 스택
- HTML5
- CSS3 (CSS Variables, Flexbox, Grid)
- Vanilla JavaScript (ES6+)
- AOS Library (Animate On Scroll)

## 반응형 디자인
- Desktop: 1280px+
- Tablet: 768px ~ 1024px
- Mobile: < 768px

## 성능 요구사항
- 초기 로딩 시간: < 3초
- 이미지 레이지 로딩
- CSS/JS 최적화
- 접근성 (WCAG 2.1 AA)

### Phase 1: 기획 및 설정 (Week 1)
- 프로젝트 초기화
- Git/GitHub 설정
- 디자인 시스템 구축
- 파일 구조 생성

### Phase 2: 기본 구조 개발 (Week 1-2)
- HTML 구조 작성
- CSS 변수 정의
- 네비게이션 구현
- Hero 섹션 구현

### Phase 3: 주요 섹션 개발 (Week 2-3)
- About 섹션
- Services 섹션
- Portfolio 섹션
- Achievements 섹션
- Contact 섹션

### Phase 4: 인터랙션 및 애니메이션 (Week 3)

- 카운터 애니메이션

- 카드 틸트 효과

### Phase 5: 반응형 및 최적화 (Week 4)
- 모바일 반응형
- 태블릿 반응형

- 접근성 개선

### Phase 6: 테스트 및 배포 (Week 4)
- 크로스 브라우저 테스트
